# PrettyFly Gig Deployment Engine — Design Spec

> Generated by Jr Dev Agentic Workflow (Phase 2: Design)
> PIVOT: Document-driven system, not a web application.

## System Architecture

```
Strategy Engine + Content Factory
├── strategy-matrix.md          ← Master grid: services x platforms x pricing
├── generator/                  ← Claude Code agent for content generation
│   ├── gig-generator.md        ← Agent: strategy row → platform-ready listing
│   └── proposal-generator.md   ← Agent: job posting + template → custom proposal
├── listings/                   ← Generated platform-ready content
│   ├── fiverr/                 ← Fiverr-formatted gig listings
│   ├── upwork/                 ← Upwork profile sections + gig descriptions
│   ├── toptal/                 ← Toptal profile positioning
│   └── arc/                    ← Arc.dev profile positioning
├── tracker.md                  ← What's live, performance data
├── checklists/                 ← Per-platform posting guides
│   ├── fiverr-checklist.md
│   ├── upwork-checklist.md
│   ├── toptal-checklist.md
│   └── arc-checklist.md
└── docs/                       ← Existing strategy docs (already written)
    ├── STRATEGY.md
    ├── GIG-COPY.md
    ├── PLATFORM-GUIDES.md
    └── PROPOSAL-TEMPLATES.md
```

## Strategy Matrix Design

The core artifact. One row per listing to deploy:

| Column | Purpose |
|--------|---------|
| ID | Sequential (GIG-001, GIG-002...) |
| Pillar | Which of the 5 service pillars |
| Service | Specific service name |
| Fiverr | Pricing tiers (Basic/Standard/Premium) or "-" if not applicable |
| Upwork | Rate range or fixed price range |
| Toptal | Hourly rate or "-" |
| Arc.dev | Hourly rate or "-" |
| Priority | P1 (launch first) / P2 / P3 |
| Status | draft / generated / posted / live |
| Keywords | SEO/search terms for platform discoverability |

Target: 20-30 rows covering all 5 pillars across relevant platforms.

## Content Generator Agent Design

### Gig Generator (`~/.claude/agents/gig-generator.md`)

**Input**: A row from the strategy matrix + platform target
**Output**: Platform-ready listing content in the correct format

**Fiverr output format:**
- Title (80 chars max, "I will..." format)
- Category + subcategory
- Search tags (5)
- 3-tier pricing table (name, description, delivery days, revisions, price)
- Gig description (structured: hook, deliverables, why me, process, FAQ, CTA)
- Requirements (what buyer provides before starting)
- FAQ (5 questions)

**Upwork output format:**
- Specialized Profile section (title, overview paragraph, skills list)
- Service description for profile
- Sample proposal for this service type
- Suggested hourly rate or fixed-price range
- Portfolio case study outline

**Toptal/Arc.dev output format:**
- Profile headline
- Summary paragraph (enterprise-focused)
- Key skills and technologies
- Rate positioning rationale

### Proposal Generator (enhance existing `docs/PROPOSAL-TEMPLATES.md`)

**Input**: Job posting URL or description + template selection
**Output**: Custom proposal under 200 words following the HOOK-CREDIBILITY-APPROACH-DIFFERENTIATOR-CTA framework

## Platform Checklist Design

Each platform gets a step-by-step posting checklist:

**Fiverr Checklist:**
1. Log in to fiverr.com/seller_dashboard
2. Click "Create a New Gig"
3. Title: [paste from generated content]
4. Category: [specified in generated content]
5. Search Tags: [paste 5 tags]
6. Pricing Table: [fill 3 tiers from generated content]
7. Description: [paste formatted description]
8. Requirements: [paste requirements]
9. Gallery: Upload gig image (create in Canva using template)
10. FAQ: [paste 5 FAQs]
11. Publish → Update tracker.md with URL and date

**Upwork Checklist:**
1. Log in to upwork.com
2. Go to Profile → Specialized Profiles
3. Create/update profile for this service pillar
4. Paste headline, overview, skills from generated content
5. Add portfolio item if available
6. Set hourly rate per generated content
7. Update tracker.md

## Phase 2 Consideration: When to Build the App

Build a lightweight web app ONLY when:
- 20+ listings are live and generating data
- Manual tracking in markdown becomes a bottleneck
- Revenue data needs visualization for optimization decisions
- The original Gig-Engine scaffold can be stripped down and reused

Until then, the document system + agents is faster, cheaper, and more flexible.
